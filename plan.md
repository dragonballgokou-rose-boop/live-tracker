## 詳細表示機能の実装方針

### 1. ライブ詳細の表示
- **対象箇所**:
  - ダッシュボードの「日付別 参加スケジュール」内のライブ名（`.date-event-name`）
  - 集計表（Tally）の左列のライブ名 / 日程（`.tally-table td:first-child`）
- **動作**: 
  - 上記の要素をクリック可能にし（cursor: pointer等）、クリック時に `showLiveDetailsModal(liveId)` を呼び出す。
  - **表示内容**:
    - ライブ名、アーティスト、開催地、期間、メモ
    - **参戦予定メンバーの一覧**（参戦・未定の振り分け）
    - UIには既存の `showModal` および `escapeHtml` などのユーティリティを活用する。

### 2. メンバー詳細（参戦予定ライブ一覧）の表示
- **対象箇所**:
  - ダッシュボードの「メンバー別参戦数」のメンバー名（`.ranking-name` 等）
  - 集計表のヘッダーにあるメンバーアイコン（`.tally-table th` の中身）
  - メンバー管理画面のカード（`.member-card`）
- **動作**:
  - 対象要素をクリック時、`showMemberDetailsModal(memberId)` を呼び出す。
  - **表示内容**:
    - メンバー名（アイコン含む）
    - 全体の参戦チケット数 / 参戦率
    - **そのメンバーが「参戦」または「未定」にしているライブの一覧**（日付順）

### 3. ローカル（ポート5173）で日程が直っていない件について
- **原因・方針**:
  - Viteのキャッシュ（ブラウザのハードリフレッシュが必要）か、あるいは `npm run dev` で起動している間にファイルのホットリロードが不完全だった可能性。
  - あるいは、**古いデータ自体が `localStorage` やGAS上に1日ズレた状態で保存され続けている**のが原因の可能性が最も高い。（コードを直しても、データベースの中身がズレたままになっている）
  - **対応策**: データイニシャライズ（再計算・再保存）処理を走らせるか、ユーザーに一旦「全編集内容を保存（または再登録）」してもらい正しい日付形式でDBを上書き更新してもらうよう案内する。（今回はJSが YYYY-MM-DD で上書きすれば直る）
